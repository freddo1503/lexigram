= AWS Infrastructure
:description: CDK stack: Docker Lambda, DynamoDB table, EventBridge cron, IAM, Secrets Manager.

Infrastructure is defined in `infra/` using AWS CDK (Python).

== Stack: Lexigram

`infra/lexigram_stack.py` -- single CDK stack.

=== Components

[%header,cols="1,2"]
|===
|Resource |Configuration

|*DynamoDB Table*
|`LawPostsDynamoDBTable` -- partition key `textId` (STRING), on-demand billing, DESTROY removal policy

|*Secrets Manager*
|`aws_secretsmanager.Secret.from_secret_name_v2` -- imports the existing `my-env-secrets` by name (read-only reference, never mutated by CDK)

|*Lambda Function*
|`DockerImageFunction` -- Python 3.12 base image, 512 MB memory, 10-minute timeout, 1-month log retention

|*EventBridge Rule*
|`cron(30 11 ? * MON-FRI)` -- weekdays at 11:30 UTC

|*IAM*
|Lambda -> DynamoDB (read/write), Lambda -> Secrets Manager (GetSecretValue)
|===

=== Lambda Configuration

* *Architecture:* Auto-detected (ARM64 or X86_64 based on build machine via `platform.machine()`)
* *Handler:* `app.handler.handler`
* *Docker:* Built from project root `Dockerfile`

=== Secrets Bootstrap

The `my-env-secrets` secret must exist in AWS Secrets Manager (region `eu-west-3`) *before* `cdk deploy` runs. CDK imports it by name and grants the Lambda execution role `secretsmanager:GetSecretValue` -- it never creates, updates, or reads the secret values during deploy.

Populate or rotate the secret out-of-band, for example:

[source,bash]
----
aws secretsmanager put-secret-value \
  --secret-id my-env-secrets \
  --secret-string file://secrets.json
----

`secrets.json` is a flat JSON object keyed by field name (e.g. `MISTRAL_API_KEY`, `LEGIFRANCE_CLIENT_ID`, `LEGIFRANCE_CLIENT_SECRET`, `INSTAGRAM_ACCESS_TOKEN`, `INSTAGRAM_APP_SECRET`). `AWSSecretsManagerSource` in `app/config.py` reads this JSON at Lambda cold start.

== CDK App Entry

`infra/app.py` (17 lines):

* Calls `get_account_and_region()` from `infra/helpers.py`
* Creates the `Lexigram` stack with the resolved AWS account and region
* Region: `eu-west-3` (Paris)

== Infrastructure Helpers

`infra/helpers.py` (53 lines):

`get_account_and_region(profile=None)` supports two modes:

. *AWS Profile:* reads `AWS_PROFILE` env var, uses boto3 Session
. *Direct credentials:* reads `AWS_ACCESS_KEY_ID`, `AWS_SECRET_ACCESS_KEY`, `CDK_DEFAULT_REGION`
